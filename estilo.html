Aula 06 - Mão na Massa

Entenda a Diferença

Git

Ferramenta (software) no seu PC para gerenciar versões.

Funciona offline.

GitHub

Nuvem (rede social de código) para hospedar os repositórios.

Permite colaboração online.

O Início Local

A engenharia de requisitos nos ensinou a ter ordem. O Git faz o mesmo com o código.

# Cria a pasta e entra nela
$ mkdir minha-livraria-online
$ cd minha-livraria-online

# Inicia o rastreamento (Cria a pasta .git)
$ git init

NÍVEL 2: Os 3 Estados do Git

Acompanhe o fluxo da carta:

  1. Modified (Working Directory): Carta sendo escrita no papel.
  2. Staged (Staging Area): Carta no envelope e selada.
  3. Committed (Repository): Carta na caixa do correio. É histórico.

💻 Ação Local #1: Criar e Salvar Versão

Siga este fluxo no terminal na pasta do projeto:

# 1. Crie um arquivo simples
$ echo "Sistema de Livraria" > readme.md

# 2. Veja o estado (Modified)
$ git status

# 3. Prepare o arquivo (Staged)
$ git add readme.md

# 4. Salve a versão (Committed)
$ git commit -m "Adiciona readme inicial"

O histórico é o seu currículo

Você criou a primeira "foto" (snapshot) do projeto.

# Lista todos os commits feitos
$ git log

# Versão resumida
$ git log --oneline

A Engineering of Requirements gerou a documentação, o Git gera a prova do trabalho.

NÍVEL 3: Conexão com o GitHub

Seu código está a salvo no seu PC, mas e se o PC quebrar? Vamos para a nuvem.

  1. Crie uma conta/login no GitHub (github.com).
  2. Clique em "New Repository" (Novo Repositório).
  3. Dê o nome exato: minha-livraria-online.
  4. Não marque nenhuma opção extra (README, .gitignore). Clique em Create.

💻 Ação Remota #1: Enviar o Código (Push)

Copie os comandos que o GitHub te mostrou (em 'Push an existing repository'):

# 1. Conecta o PC ao servidor remoto (Troque 'usuario' pelo seu)
$ git remote add origin https://github.com/usuario/minha-livraria-online.git

# 2. Renomeia branch principal (padrão atual)
$ git branch -M main

# 3. Empurra o código (Upload)
$ git push -u origin main

NÍVEL 4: Sincronização e Equipe

Seu código está online. Como um colega trabalha nele?

Puxar (Pull)

Baixar atualizações que estão no GitHub para o seu PC.

git pull origin main

Clonar (Clone)

Baixar um projeto completo do GitHub pela primeira vez.

git clone https://www.modoprojetos.com.br/

Desafio de Integração

  1. Vão para a pasta do projeto (minha-livraria-online).
  2. Crie um arquivo requisitos.txt e cole 2 Requisitos Funcionais que vocês definiram na atividade anterior.
  3. Siga o fluxo: git addrequisitos.txt -> git commit -m "Adiciona requisitos do sistema".
  4. Dê o git push origin main.
  5. Confirme no GitHub se o arquivo apareceu lá.
